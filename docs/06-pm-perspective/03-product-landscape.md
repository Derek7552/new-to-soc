# SOC 产品版图与竞争格局

> 面向：产品经理、产品负责人、战略规划者 — 需要理解"谁在做什么、我在哪个位置、机会在哪"
> 解决的核心决策问题：产品应该定位在哪个品类？对标谁？差异化怎么打？

## TL;DR

SOC 产品赛道分为五大品类：SIEM（日志中枢）、EDR/XDR（终端检测）、SOAR（自动化编排）、MDR/MSSP（托管服务）、AI 原生 SOC（Agent 驱动）。赛道正经历三级跳：工具拼装 → 平台整合 → AI Agent 替代人力。每一跳都在改变竞争规则——上一跳的领先者在下一跳可能成为被颠覆者。产品定位的核心不是"做更多功能"，而是"你在哪一跳、在哪一细分市场、解决了谁的什么不可替代需求"。

## 五大品类总览

```
                        AI 原生 SOC（新兴）
                        Dropzone / Exaforce
                        Torq / Prophet / D3
                              ↑
                              │ 范式迁移：AI Agent 替代人力
                              │
    ┌─────────┬───────────────┼───────────────┬─────────┐
    │         │               │               │         │
    │  SIEM   │    EDR/XDR    │    SOAR       │  MDR    │
    │  日志中枢 │    终端检测    │   自动化编排   │ 托管服务  │
    │         │               │               │         │
    │ Splunk  │  CrowdStrike  │  XSOAR        │  MSSP   │
    │ Sentinel│  Sentinel     │  Torq         │  MDR    │

    └─────────┴───────────────┴───────────────┴─────────┘
```

## 品类 1：SIEM（安全信息与事件管理）

### 产品本质

**集中收集所有安全日志 → 关联分析 → 产生告警。** 是 SOC 的"数据中枢"。

### 代表厂商与定位

| 产品 | 定位 | 典型客户 | 定价模式 |
|------|------|---------|---------|
| **Splunk Enterprise Security**（Cisco） | 最强大查询生态，SPL 是事实标准 | 大型企业、政府 | 按数据摄入量，$100K+ 起 |
| **Microsoft Sentinel** | 云原生、Azure/M365 生态深度绑定 | Microsoft 全家桶客户 | 按数据量 + 分析量 |
| **Google Chronicle** | 无限数据留存、大数据分析 | 数据量巨大的大型企业 | 按用户数 |
| **Elastic Security** | 开源基因、灵活部署 | 技术公司、中型企业 | 开源免费 / 企业订阅 |
| **IBM QRadar** | 老牌 SIEM、关联分析引擎强 | 金融、政府 | 按 EPS（每秒事件数） |

### 竞争要点

- **迁移成本极高**：换了 SIEM = 所有日志管道重做、所有检测规则重写、所有分析师重新学查询语言
- **SPL 是护城河**：Splunk 生态的查询语言 SPL 已成为行业技能标准，分析师不愿意学新的
- **数据驻留与合规**：日志数据涉及隐私/合规，很多企业要求本地化部署或指定区域存储
- **存量市场增速放缓**：大企业基本都有 SIEM，增长来自存量升级和数据量膨胀，而非新客户

### 对 PM 的启示

如果你想做 SIEM，问自己：**SPL 生态怎么绕？数据迁移成本怎么降？** 如果你的产品依赖 SIEM 但自己不做，问自己：**支持哪些 SIEM 的 API 集成？数据不够时怎么办？**

## 品类 2：EDR / XDR（终端/扩展检测与响应）

### 产品本质

**在终端上实时监控行为 → 检测恶意活动 → 提供响应能力。** XDR 是 EDR 的扩展版，把检测范围从终端延伸到网络、云、邮件、身份。

### 代表厂商

| 产品 | 定位 | 关键优势 |
|------|------|---------|
| **CrowdStrike Falcon** | EDR 标杆，轻量 Agent、云端分析 | 终端市场占有率第一，Charlotte AI |
| **Microsoft Defender for Endpoint** | Windows 原生绑定 | M365 E5 打包销售，边际成本极低 |
| **Palo Alto Cortex XDR** | XDR 品类定义者 | 跨终端+网络+云的统一检测 |
| **SentinelOne** | AI 驱动的自主响应 | Singularity 平台，强调自动修复 |

### 竞争要点

- **Agent 是护城河也是负担**：部署 Agent 需要数月，但一旦部署就极难替换（全网重装 + 重启）
- **Microsoft "免费"策略**：Defender 绑定 M365 授权，对已付费客户来说是零边际成本——纯 EDR 厂商需要用差异化对抗"免费"
- **从 EDR 到 XDR 到平台**：CrowdStrike 和 Palo Alto 都在向上整合 SIEM + SOAR + ASM，做全栈平台

### 对 PM 的启示

- 做 EDR 需要**终端 Agent 开发能力** + **海量终端管理稳定性** + **对操作系统内核级的理解**——技术门槛极高
- 但如果你的产品不需要做终端，那么**支持哪些 EDR 的 API 集成**就是关键选型

---

## 品类 3：SOAR（安全编排、自动化与响应）

### 产品本质

**把重复性的告警处置流程自动化执行。** 比如：收到钓鱼告警 → 自动查发件域信誉 → 自动拉黑 → 自动搜索其他收件人 → 自动通知用户。

### 代表厂商

| 产品 | 定位 | 关键优势 |
|------|------|---------|
| **Palo Alto Cortex XSOAR**（Demisto） | 品类定义者，市场占有率第一 | 1300+ Playbook、1100+ 集成 |
| **Splunk SOAR**（Phantom） | 与 Splunk ES 绑定 | 与 SIEM 原生集成 |
| **Torq** | 新一代 SOAR，AI 原生 | 无代码、自然语言驱动 |
| **Swimlane** | 中端市场 SOAR | 价格更友好 |
| **Tines** | 轻量自动化，Story 模型 | 极简部署、适合小团队 |

### 竞争要点

- **Playbook 维护是隐性成本**：写 Playbook 要安全专家 + 运营经验，维护起来工作量不低——这是一条隐形护城河
- **自然语言替代 Playbook**：新一代产品（如 Dropzone、Torq HyperAgents）用自然语言描述处置逻辑，降低维护门槛
- **集成数量不是越多越好**：1100+ 集成听起来厉害，但 90% 的客户只用其中的 15 个

### 对 PM 的启示

- SOAR 是典型的"买了不一定用"产品——大量企业买了 XSOAR 但只跑 3-5 条简单 Playbook
- **激活率比功能数重要**：一个能让客户 1 小时内跑起第一条自动化规则的产品，比一个能写 1000 种 Playbook 但需要 3 个月部署的产品更有商业价值

---

## 品类 4：MDR / MSSP（托管检测与响应 / 安全托管服务）

### 产品本质

**不是卖软件，是卖"人+平台"的运营服务。** 客户把安全监控外包给 MSSP，MSSP 用自己的人员和平台 7×24 值守。

### 代表厂商

| 类型 | 代表 | 模式 |
|------|------|------|
| 纯 MDR 厂商 | Arctic Wolf、Red Canary、eSentire | 用自己的平台 + 自己的分析师 |
| 平台厂商 MDR 服务 | CrowdStrike Falcon Complete、Microsoft Defender Experts | 用自家产品 + 外包运营 |
| 本土 MSSP | 安恒、奇安信、深信服（中国） | 产品 + 驻场 + 远程运营 |
| 电信/运营商 MSSP | Verizon、AT&T、NTT | 捆绑网络服务 |

### 竞争要点

- **人是成本中心**：MDR 毛利受限于分析师薪资，规模化需要 AI 替代 Tier 1
- **信任比能力重要**：客户要把所有安全日志给你看，信任门槛极高
- **MSSP 正在成为 AI SOC 平台的核心渠道**：Dropzone AI 等产品大量通过 MSSP 触达中小客户

### 对 PM 的启示

- 如果你的产品想做 MSSP 渠道，核心卖点是**"让你的分析师每人多处理 3-5 倍告警"**
- MSSP 客户的忠诚度比直销客户更高——他们把你的产品嵌入了自己的服务流程，替换成本极高

---

## 品类 5：AI 原生 SOC（Agentic SOC）— 新兴品类

### 产品本质

**不是"加了 AI 功能的传统工具"，而是从架构起点就以 AI Agent 为核心组织单元。** 详见 [AI 原生 SOC 深度解析](04-ai-soc-deep-dive.md)。

### 代表厂商速览

| 产品 | 核心定位 | 起始价格 | 特色 |
|------|---------|---------|------|
| **Dropzone AI** | AI SOC Analyst，替代 Tier 1 | $36K/年起 | 玻璃盒推理、90+ 集成、1 小时部署 |
| **Exaforce** | 任务专属 Agent + 双模式 | 未公开 | 自动驾驶 + 副驾驶 |
| **Torq HyperAgents** | AI Agent + 自动化 | 未公开 | 可执行响应动作 |
| **Prophet Security** | 端到端调查 + 响应 | 未公开 | 全自动化调查 |
| **D3 Morpheus** | 全生命周期 Agentic SOC | 未公开 | Flat-rate 定价 |
| **Radiant Security** | 中小企业 AI SOC | $1K+ 起 | 定价极低 |

### 竞争要点

- **不存数据的架构**：AI 原生平台通过 API 联邦查询现有 SIEM/EDR，省去了数据湖成本
- **按调查次数定价**：与传统 SIEM 按数据量定价形成鲜明差异，但也带来"挑拣告警"的风险
- **模块化入场**：从 Tier 1 替代开始，逐步扩展到 Threat Hunter、Intel Analyst——降低客户试错成本
- **与传统巨头的关系**：目前是互补（AI Agent 装在传统平台之上），长期可能替代或整合

## 大厂 vs 创业公司：不同打法的取舍

| 维度 | 大厂（Palo Alto、CrowdStrike、Microsoft） | 创业公司（Dropzone、Exaforce、Torq） |
|------|------------------------------------------|--------------------------------------|
| **产品策略** | 全栈一体化，让客户"用我一家就够了" | 单点极致，做最擅长的一层 |
| **技术路线** | 自建数据湖 + 闭源模型 | API 联邦 + LLM 驱动 |
| **定价** | $100K-$1M+/年 | $12K-$100K/年 |
| **部署周期** | 数周到数月 | 数小时到数天 |
| **目标客户** | 大型企业、政府 | 中型企业、MSSP |
| **销售方式** | 直销 + 渠道 + 专业服务 | PLG + MSSP 渠道 + 低接触销售 |
| **AI 策略** | AI 嫁接既有平台（AgentiX、Charlotte AI） | AI 是原生架构起点 |

### 混合架构是现实

纯"平台派"和纯"AI 原生派"的二分法在实际部署中越来越模糊。很多企业实际是：

```
Cortex / Splunk / Sentinel 做数据底座
         +
Dropzone / Exaforce 做 AI 分析师层
         +
内部自研的胶水自动化脚本
```

→ **产品启示**：不要妄想"我一家包办全部"，思考"我在栈的哪一层不可替代"

## 竞争格局判断框架

面对任何一个竞品或潜在竞品，用四个问题快速定位：

1. **它在哪一跳？** — 工具/平台/AI Agent？同一个市场阶段是直接竞争，跨阶段则可能是"先互补后竞争"
2. **它切的是哪个用户的分层？** — CISO 视角 / 分析师视角 / IT 运营视角？同用户层直接抢注意力
3. **它的钱从哪来？** — 按数据量/按用户/按告警量/按服务人天？定价模式冲突 = 商业模式冲突
4. **它的护城河是什么？** — 数据积累/Agent 安装量/查询语言生态/Playbook 库/合规认证？护城河是否有网络效应？

---

## 对产品决策者的关键启示

1. **品类归属决定了客户的比价对象**：自称为"AI SOC 平台"会被对比 Dropzone，自称为"XDR"会被对比 CrowdStrike——想清楚你希望客户把你看作谁的替代品
2. **不要在功能数量上与大厂正面竞争**：大厂有 1000+ 集成和 1300+ Playbook，创业公司的机会在"更简单、更快、更便宜"
3. **Gartner 首次认证 AI SOC Agents 品类（2025）**：这意味着企业客户的采购单上开始单独列出这个品类——是窗口期也是竞争加剧的信号
4. **MCP（Model Context Protocol）可能改变竞争规则**：如果 AI Agent 可以跨厂商协作，那么"全栈一体化"的锁定策略会受到挑战
